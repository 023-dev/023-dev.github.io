# 데이터베이스의 개념과 특징

현대 사회는 데이터를 활용한 정보의 생성과 관리가 매우 중요한 시대입니다. 이에 따라 데이터베이스(Database)는 대량의 데이터를 체계적으로 관리하고 활용하기 위한 핵심 도구로 자리잡았습니다. 이번 글에서는 데이터베이스의 역할, 파일 시스템과의 차이점, 특징, 그리고 DBMS(Database Management System)에 대한 개념과 특성을 살펴보겠습니다.

---

## 파일 시스템과 데이터베이스의 차이점

### 파일 처리 시스템
파일 처리 시스템은 초기 데이터 관리 방식으로, 개별 애플리케이션이 데이터를 독립적으로 관리하는 구조입니다. 이는 특정 하드웨어나 소프트웨어에 종속되며, 다음과 같은 문제점을 야기합니다:

1. **데이터 종속성**: 특정 환경에 맞춘 데이터 구조로 인해 데이터의 독립적인 재사용이 어렵습니다.
    - 물리적 종속성: 저장 매체 변경 시 데이터 변경 필요.
    - 논리적 종속성: 데이터 구조 변경 시 애플리케이션 수정 필요.

2. **데이터 중복성**: 동일 데이터가 여러 파일에 저장되며, 이는 데이터 일관성 문제를 유발합니다.
    - 데이터의 무결성 훼손 가능성.
    - 데이터 저장 공간의 비효율적 사용.

3. **무결성 및 동시성 문제**: 데이터의 제약 조건을 시스템 차원에서 보장하지 않으며, 다수 사용자의 동시 접근 시 충돌 위험.

### 데이터베이스 시스템
반면, 데이터베이스 시스템은 데이터 저장 및 관리를 위한 중앙화된 구조를 제공합니다. 데이터베이스는 데이터 중복과 종속 문제를 해결하며, 무결성과 동시성을 보장합니다. 주요 차이점은 다음과 같습니다:

- **데이터 독립성**: 물리적 저장 구조와 논리적 데이터 구조가 분리.
- **일관성 및 무결성 보장**: 제약 조건을 설정하여 데이터의 신뢰성을 확보.
- **효율적 검색**: 최적화된 쿼리 처리로 대량 데이터를 빠르게 검색 가능.

---

## 데이터베이스의 특징

### 데이터베이스의 주요 특징
1. **자기 기술성**
    - 데이터베이스는 데이터와 데이터 구조(메타데이터)를 포함하여 자체적으로 정의.

2. **프로그램과 데이터의 격리**
    - 데이터베이스 구조 변경이 애플리케이션에 미치는 영향을 최소화.

3. **다중 뷰 제공**
    - 다양한 사용자 요구에 따라 데이터의 부분적 표현 가능.

4. **동시성 및 공유**
    - 다수 사용자가 동시에 데이터를 수정하거나 검색하더라도 일관성을 유지.

---

## DBMS란?

DBMS(Database Management System)는 데이터베이스를 관리하기 위한 소프트웨어입니다. DBMS의 주요 특징은 다음과 같습니다:

1. **데이터 보안성 보장**
    - 권한 관리 및 암호화를 통해 데이터를 보호.

2. **무결성 유지**
    - 데이터 제약 조건을 설정하여 정확성과 일관성 보장.

3. **트랜잭션 관리**
    - ACID(Atomicity, Consistency, Isolation, Durability) 속성을 통해 안정적 데이터 처리.

4. **효율적 데이터 접근**
    - 최적화된 검색과 인덱싱을 지원.

## 스키마와 데이터 독립성

### 스키마
스키마는 데이터베이스 구조를 정의한 틀입니다. 데이터베이스의 3단계 구조는 다음과 같습니다:

1. **외부 단계**
    - 사용자 관점에서의 데이터 뷰를 정의.

2. **개념 단계**
    - 데이터베이스 전체 구조를 기술하며, 데이터 무결성과 보안을 관리.

3. **내부 단계**
    - 데이터의 물리적 저장 구조를 정의.

### 데이터 독립성
데이터 독립성은 데이터베이스의 구조 변경이 애플리케이션에 영향을 미치지 않도록 보장합니다:

1. **물리적 독립성**: 물리적 저장 구조 변경이 논리적 데이터 구조에 영향을 주지 않음.
2. **논리적 독립성**: 논리적 데이터 구조 변경이 사용자 응용 프로그램에 영향을 주지 않음.

## 관계형 데이터베이스(RDBMS)

### RDBMS란?
RDBMS(Relational Database Management System)는 데이터를 테이블 형식으로 관리하는 시스템입니다. 주요 특징:

1. **릴레이션 기반**
    - 데이터는 행(Row)과 열(Column)로 구성된 릴레이션(Relation)에 저장.

2. **SQL 지원**
    - 구조화된 질의 언어(SQL)를 통해 데이터 조작 및 질의 가능.

### 릴레이션의 주요 개념
1. **스키마**: 테이블 구조를 정의하는 틀.
2. **인스턴스**: 테이블에 저장된 실제 데이터.
3. **차수(Degree)**: 테이블의 열(Column) 개수.
4. **카디널리티(Cardinality)**: 테이블의 행(Row) 개수.

---

## 키(Key)의 개념

데이터베이스에서 키는 데이터를 고유하게 식별하기 위한 속성입니다. 주요 키의 종류:

1. **슈퍼키(Super Key)**
    - 데이터베이스의 모든 행을 유일하게 식별할 수 있는 속성 집합.

2. **후보키(Candidate Key)**
    - 최소한의 속성으로 이루어진 슈퍼키.

3. **기본키(Primary Key)**
    - 후보키 중 하나를 선택하여 테이블의 주 키로 설정.

4. **대리키(Surrogate Key)**
    - 시스템이 생성한 인공 키.

5. **외래키(Foreign Key)**
    - 다른 테이블의 기본키를 참조하는 키로, 테이블 간 관계를 정의.

---

## 무결성 제약조건

데이터 무결성을 보장하기 위한 제약조건은 다음과 같습니다:

1. **도메인 무결성**
    - 속성 값의 유효성을 보장.

2. **개체 무결성**
    - 기본키 값은 Null이거나 중복될 수 없음.

3. **참조 무결성**
    - 외래키 값은 참조되는 기본키 값과 일치하거나 Null이어야 함.

## 사용 경험과 MySQL 설명

### MySQL 개요
MySQL은 오픈 소스 RDBMS로, 높은 성능과 안정성을 제공합니다. 주요 엔진은 다음과 같습니다:

1. **InnoDB**
    - 트랜잭션 지원 및 외래키 제약조건 제공.
    - 데이터 무결성 보장과 동시성 제어.

2. **MyISAM**
    - 읽기 작업이 많은 환경에 적합.

### InnoDB 특징
InnoDB는 ACID 트랜잭션을 지원하며, 데이터 무결성과 동시성을 보장합니다. 인덱싱과 외래키를 통해 관계형 데이터베이스의 강점을 극대화합니다.
